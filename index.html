A reading group at Stellenbosch starting 5 March 2025 for the book The Little Typer

The Little Typer provides an introduction to the field of writing programs that are proofs. This includes writing proofs with Proof Assistants and programming in Dependently Typed Programming Languages like Lean, Agda and Coq. These tools allow us to not only prove that our critical code is correct, but also to write mathematical proofs in a way in which we can collaborate with other authors on github. Examples industry use cases and mathematical proofs

  • Four Color Theorem
  • Formalized Web Assembly Spec
  • Cryptographic Arithmetic for Chrome and Firefox
  • Perfectoid Spaces - Kevin Buzzard, et al.

Examples of people that might be interested

  • You are a computer science student, but struggle to find the motivation to understand mathematical proofs
  • You are a mathematics student trying to understand how their skills can be applicable for programming in industry.
  • You are a working programmer looking for a new challenge.
  • You are a Mathematician looking for a better way to collaborate.
  • Anyone with a basic understanding of recursion.

Requirements

You do not need to be an experienced programmer or be a able to write or read mathematical proofs. You do need to be able to write code that uses recursion. For example, if you at some stage were able to write a sorting algorithm using recursion or you can now write a program to calculate the length of a linked list using recursion, then you meet the requirements. If you have not done this before we recommend reading The Little Schemer first.

Schedule

Each Wednesday starting 5 March 2025 we will discuss chapters we have read and exercises we completed for that week. The schedule is quite intense. It is not a lot to read, but it is a lot to grasp and each week has exercises. We estimate that this will be about 2 to 5 hours of work per week for 10 weeks.
